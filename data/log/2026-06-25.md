date: 2026-06-25
title: Typing code, one frame at a time
---
Code Cap is a CLI tool that turns source code into MP4 typing animations — point
it at a file and a line range, and it renders those lines being typed out
character-by-character with syntax highlighting. The output is ready to drop
into a TikTok, a YouTube short, or an Instagram reel. It's built on Remotion and
Shiki, which handle the video rendering and syntax highlighting respectively.
The main script loops through clips, renders each one headlessly, then hands the
results off to ffmpeg for concatenation.

The bulk of today was refactoring the render pipeline to support a compiled
binary distribution. Remotion relies on native components — a Rust-based
compositor and headless Chrome — so there's no clean way to bundle everything
into a single self-contained executable. The approach instead is to pre-bundle
the Remotion composition at build time, embed it as a tarball, and extract it at
runtime next to the binary. It's a bit of a workaround, but it means the binary
can be distributed without requiring users to have Bun or the node_modules tree
around. The render script now detects whether it's running as a compiled binary
and adjusts its bundle path accordingly.

The CLI already supports three video presets (TikTok 9:16, YouTube 16:9,
Instagram 1:1), per-clip typing speed overrides, and a project-level config file
for setting defaults. Next up is polishing the font size auto-fit logic and
testing the full binary build on a clean machine.
